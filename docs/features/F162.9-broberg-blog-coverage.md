# F162.9 — Expand broberg Gate A coverage to blog posts + classify 27 surfaced fields

## Motivation

F162.8 corrected broberg's coverage route list: it previously listed 12 BARE top-level slugs (`/inline-redigering`, `/bevis-ikke-loefter`, …) that resolve to a generic placeholder — the real, fully-editable articles live at `/{category}/{slug}` (e.g. `/indsigter/bevis-ikke-loefter`). The content was never missing; the URLs were wrong. But the correction revealed the gate scans only 8 static marketing pages — **the ~17 DA blog posts (the site's richest editable content) are not covered at all.**

Adding them proves out clean on A.1 (`cms check-editable`): 25 pages, **1399 editable fields, 0 visible-text gaps** — visitors can edit everything they see. But A.2 (`cms coverage`, schema-level) surfaces **27 pre-existing fields** not wired for inline-edit. These are not regressions (the pages were simply never scanned); they deserve a proper wire/classify pass, not a rushed baseline dump — hence this follow-up story rather than inlining it into F162.8.

## The 27 fields (from `bun run gate:coverage` on the 25-page set)

- **`categories/*: accent`** (×5) — a color token → add `accent` to the A.2 `--ignore` list (structural, like the existing ignores).
- **`posts/*: excerpt`** (×7) — the listing/summary field. A.1 confirms it is NOT rendered as visible prose on the detail page (it's the card summary / SEO) → add `excerpt` to `--ignore`.
- **`categories/*: description`** (×5) — renders on the category INDEX pages (not scanned), a label on post pages → baseline or ignore with reason.
- **`blocks/*-case-* + *-comparison: caption, text`** (×10 across sanne-case-*, xrt-case-*, design-altitude-comparison) — custom case-study block prose. A.1 misses these because they render inside divs/spans, NOT the CONTENT_SEL (p/h/li/…). **These are the genuine question:** are they visible prose a visitor reads (→ WIRE `data-cms-field`) or structural captions (→ baseline)? Inspect the case-block components in broberg `src/components` and decide per field.

## Scope

1. Add DA blog-post URLs to `scripts/lib/public-routes.mjs` PUBLIC_ROUTES (17 verified). Decide EN twins + `/{category}` index pages.
2. `--ignore` += `accent,excerpt` in `scripts/gate-coverage.mjs`.
3. Inspect the 10 case-block fields → wire the visible ones, baseline the structural ones (documented).
4. Baseline `categories/*/description` (or ignore) with reason.
5. Both gates green on the expanded set.

## Non-goals

- Changing A.1's CONTENT_SEL to catch div/span text (would flood false positives on layout text) — the fix is wiring the specific block fields, not broadening the scan.
- Writing any NEW content — all blog content already exists (this is a coverage + wiring pass, not authoring).

## Verification

`bun run gate:editable` 0 gaps + `bun run gate:coverage` green on the expanded route set; broberg CI green on main.
