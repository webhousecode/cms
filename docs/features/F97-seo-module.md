# F97 — SEO Module

> Dedicated SEO management in the CMS admin — per-document SEO fields, keyword tracking, AI optimization, SEO audit dashboard, and structured data templates.

## Problem

SEO is currently scattered across the CMS with no unified editor experience:

- `cms ai seo` CLI command writes `_seo` fields but there's **no UI to view or edit them**
- Redaktører kan ikke selv tilføje meta title, description, keywords, eller OG image
- Ingen SEO score/audit per dokument
- Ingen keyword-tracking (hvilke keywords targeter vi, og dækker vi dem?)
- JSON-LD structured data kræver kode — redaktører kan ikke tilføje det
- F89 (Post-Build Enrichment) injicerer tags men kun som fallback — redaktøren har ingen kontrol
- F17 (AI Content Index) genererer llms.txt men redaktøren ser det aldrig

Konkurrerende CMS'er (Yoast/RankMath i WP, Sanity SEO pane, Contentful SEO app) giver redaktører et dedikeret SEO-panel med real-time scoring. Vi har intet.

## Solution

A first-class SEO module built into the CMS admin with three layers:

1. **Per-document SEO panel** — sidebar panel in the document editor with editable meta title, description, keywords, OG image, canonical URL, robots, and JSON-LD. Real-time SEO score.
2. **SEO Dashboard** — overview of all documents' SEO health, missing fields, keyword coverage, orphan pages.
3. **AI SEO Agent** — existing `cms ai seo` integrated into admin UI as a one-click "Optimize" button per document and bulk "Optimize All" on the dashboard.

## Related Features

| Feature | Relationship |
|---------|-------------|
| **F89 Post-Build Enrichment** | Injects SEO tags as fallback into built HTML. F97 gives redaktører **control** over what F89 injects. F89 uses F97's fields when available. |
| **F17 AI Content Index** | Generates llms.txt, sitemap, structured data. F97's keyword/description data feeds into F17's output. |
| **F16 Link Checker** | Finds broken links. F97's dashboard references link health as an SEO signal. |
| **F03 WP Migration** | Imports `_seo` fields from WP (Yoast/RankMath meta) during migration. |
| **F14 Newsletter** | Newsletter archive pages need SEO fields. F97 provides them. |
| **CMS AI `seo` agent** | Already exists in `cms-ai`. F97 wraps it in admin UI. |

## Technical Design

### 1. SEO Fields — Per-Document Data Model

```typescript
// Stored as `_seo` object in each document's data

export interface SeoFields {
  // Editable by redaktør
  metaTitle?: string;          // max 60 chars, shown in browser tab + Google
  metaDescription?: string;    // max 160 chars, shown in Google snippet
  keywords?: string[];         // target keywords for this page
  ogImage?: string;            // Open Graph image URL (from media library)
  ogTitle?: string;            // Override for social sharing title
  ogDescription?: string;      // Override for social sharing description
  canonical?: string;          // canonical URL (default: self)
  robots?: string;             // "index,follow" | "noindex,follow" | "noindex,nofollow"
  jsonLd?: Record<string, unknown>;  // custom JSON-LD (Article, Product, FAQ, etc.)

  // Auto-generated by AI / system
  score?: number;              // 0-100 SEO score
  scoreDetails?: SeoScoreDetail[];
  lastOptimized?: string;      // ISO timestamp of last AI optimization
  readability?: number;        // 0-100 readability score (Flesch-Kincaid or equivalent)
}

export interface SeoScoreDetail {
  rule: string;                // "meta-title-length", "keyword-in-title", etc.
  status: "pass" | "warn" | "fail";
  message: string;             // "Meta title is 45 chars (good: 30-60)"
}
```

### 3. SEO Score Rules

```typescript
// packages/cms-admin/src/lib/seo/score.ts

export function calculateSeoScore(doc: Document, seo: SeoFields): SeoScoreResult {
  const checks: SeoScoreDetail[] = [];

  // Title checks
  checks.push(checkMetaTitleLength(seo.metaTitle));        // 30-60 chars
  checks.push(checkMetaTitleKeyword(seo.metaTitle, seo.keywords)); // primary keyword in title
  checks.push(checkMetaTitleUnique(seo.metaTitle, allDocs)); // no duplicate titles

  // Description checks
  checks.push(checkMetaDescLength(seo.metaDescription));   // 120-160 chars
  checks.push(checkMetaDescKeyword(seo.metaDescription, seo.keywords));

  // Content checks
  checks.push(checkContentLength(doc.data.content));       // min 300 words
  checks.push(checkKeywordDensity(doc.data.content, seo.keywords)); // 1-3%
  checks.push(checkHeadingStructure(doc.data.content));    // has H2s, H3s
  checks.push(checkImageAltTags(doc.data.content));        // all images have alt
  checks.push(checkInternalLinks(doc.data.content));       // has internal links

  // Technical checks
  checks.push(checkOgImage(seo.ogImage));                  // OG image set
  checks.push(checkCanonical(seo.canonical));              // canonical set or default
  checks.push(checkSlugKeyword(doc.slug, seo.keywords));   // keyword in URL slug

  const score = Math.round(
    (checks.filter(c => c.status === "pass").length / checks.length) * 100
  );

  return { score, details: checks };
}
```

### 5. AI SEO Integration

The existing `cms ai seo` agent generates `_seo` fields. F97 wraps this in the admin:

```typescript
// packages/cms-admin/src/app/api/admin/seo/optimize/route.ts

// POST /api/admin/seo/optimize
// Body: { collection, slug } or { all: true }
// Calls the existing SeoAgent from @webhouse/cms-ai
// Writes _seo fields to document
// Returns updated score

// POST /api/admin/seo/suggest-keywords
// Body: { content: string }
// AI analyzes content and suggests target keywords
```

**AI capabilities in the SEO panel:**
- **Auto-generate meta title** — AI writes an SEO-optimized title from content
- **Auto-generate meta description** — AI writes a compelling 150-char snippet
- **Suggest keywords** — AI analyzes content and suggests relevant keywords
- **Suggest OG image** — AI picks the most suitable image from the document's media
- **Bulk optimize** — run AI SEO on all documents from the dashboard
- **Rewrite for keyword** — "Optimize this content for keyword X" rewrites sections

### 6. Keyword Tracker

Site-wide keyword management:

```typescript
// packages/cms-admin/src/lib/seo/keywords.ts

export interface TrackedKeyword {
  keyword: string;
  target: "primary" | "secondary" | "long-tail";
  documents: Array<{ collection: string; slug: string; density: number }>;
  totalOccurrences: number;
  coverage: number;          // % of relevant docs that include this keyword
  trend?: "up" | "down" | "stable";
}

// Stored in _data/seo-keywords.json
```

Redaktører tilføjer keywords de vil tracke. Dashboardet viser hvilke dokumenter der dækker dem og med hvilken densitet.

### 8. API Endpoints

```
# Per-document SEO
GET    /api/admin/seo/[collection]/[slug]     → get SEO fields + score
PUT    /api/admin/seo/[collection]/[slug]     → save SEO fields
POST   /api/admin/seo/optimize                → AI optimize (single or bulk)
POST   /api/admin/seo/suggest-keywords        → AI keyword suggestions from content

# Dashboard
GET    /api/admin/seo/overview                → all docs with scores, issues
GET    /api/admin/seo/keywords                → tracked keywords with coverage
POST   /api/admin/seo/keywords                → add/remove tracked keyword
GET    /api/admin/seo/report                  → export SEO report (JSON/CSV)
```
