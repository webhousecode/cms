# F162.8 — broberg double-scan adoption + globals burn-down

## Motivation

broberg's Gate A today runs only A.2 (`cms coverage`, collection-level union) — it has the blind spot F162.7 was built to close: a field rendered non-editably on a specific page can hide behind union coverage. Sanne already runs the double-scan (A.1 + A.2) and gave the improvement back. This story adopts the shared `cms check-editable` (F162.7) into broberg and uses it to burn down broberg's 7 accepted `globals` baseline gaps — confirming each is genuinely empty/structural, or wiring it and removing it from the baseline.

## Current broberg baseline (7 globals of 20 total entries)

`globals/aboutHeading` (→ `headingHtml`, HTML field, correctly excluded from inline-edit), `globals/contactCta`, `globals/contactHeading`, `globals/heroPillFallback`, `globals/heroPillLabel`, `globals/navCta`, `globals/universeCore` (→ `core`, feeds an SVG illustration label, not editable prose). Initial trace: `universeCore` and `aboutHeading` are used in `compose.ts`; `navCta`/`contactHeading`/others do not appear in `src/` render → likely unused schema fields (correct to baseline). The double-scan will mechanically confirm: a field genuinely unused/empty does not render → is not flagged; a field with content rendered non-editably IS flagged → real gap to wire.

## Scope

1. **Single-source routes** — broberg is Preact + Hono (not Next App Router), so route derivation differs from sanne's `public-routes.mjs`. broberg already enumerates ~20 routes inline in `package.json`'s `gate:coverage --pages`. Extract them into one module/file reused by both A.1 and A.2 (kill the duplicated list).
2. **A.1 step** — add `gate:editable` script (`cms check-editable --url https://broberg.ai --pages <routes> --ignore-text <token substrings>`) and a CI step in the existing post-deploy `coverage` job, running BEFORE A.2.
3. **Burn-down** — run A.1 against live broberg.ai; for every flagged element either wire `data-cms-field` (real gap) or add its text to `--ignore-text` (token field) with justification. Re-classify the 7 globals baseline entries; remove any that are actually wired.

## Non-goals

- Changing broberg's deploy mechanism (stays `flyctl deploy --local-only`).
- Editing richtext/HTML fields to be inline-editable (`aboutHeading`→`headingHtml` stays excluded — lossy, per F156/F157 scope).
- Touching sanne's repo (separate story: migrate her bespoke A.1 to `check-editable`).

## Verification

broberg CI: `gate` (Gate B) → `deploy` → `coverage` (A.1 `cms check-editable` 0 gaps, then A.2 `cms coverage` no-new-gaps) all green on main. `cms check-editable --url https://broberg.ai --pages <all>` locally → 0 gaps after burn-down.
