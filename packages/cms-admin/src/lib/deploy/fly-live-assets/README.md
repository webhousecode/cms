# Fly Live sync-endpoint

Assets that get copied into the user's project and deployed as the Fly Live
Docker image. Lives here (not in the user's repo) because cms-admin owns the
lifecycle — re-deploy infrastructure whenever the sync-endpoint is updated.

## Files

- **`server.ts`** — Bun runtime. Serves static files from `/srv/current` AND
  handles HMAC-signed `/_icd/*` endpoints. Atomic deploys via symlink swap.
- **`Dockerfile`** — oven/bun:1.1-alpine, copies `server.ts`, exposes :8080.
- **`fly.toml.template`** — Fly app config. `{{APP_NAME}}`, `{{REGION}}`,
  `{{VOLUME_NAME}}` replaced at deploy time.
- **`.dockerignore`** — excludes this README and the fly.toml template from
  the image.

## Env vars (set by Fly Secrets)

| Var | Required | Default | Purpose |
|---|---|---|---|
| `SYNC_SECRET` | yes | — | HMAC shared secret. Generated by admin on first deploy. |
| `PORT` | no | `8080` | Listen port. Leave as-is. |
| `SITE_DATA_ROOT` | no | `/srv` | Where `current/` + `deploys/` live. Mount volume here. |

## Endpoints

All `/_icd/*` endpoints require `X-CMS-Timestamp` + `X-CMS-Signature: sha256=<hmac>`
where hmac is over `${timestamp}\n${method}\n${pathWithQuery}\n${sha256(body)}`.

| Method | Path | Purpose |
|---|---|---|
| GET | `/_icd/health` | `{ ok, version }` |
| GET | `/_icd/manifest` | `{ deployId, files: { path: sha256 } }` |
| POST | `/_icd/deploys` | Begin new deploy. Returns `{ deployId }`. Clones current tree. |
| PUT | `/_icd/deploys/:id/files?path=X` | Write file (body = raw bytes) |
| DELETE | `/_icd/deploys/:id/files?path=X` | Remove file |
| POST | `/_icd/deploys/:id/commit` | Atomic symlink swap → this deploy is now current |
| DELETE | `/_icd/deploys/:id` | Abort deploy, clean up staging dir |
| POST | `/_icd/rollback` | Body `{ deployId? }`. Swap symlink back; default = previous |

## Atomic deploy model

1. `POST /_icd/deploys` → server creates `/srv/deploys/<id>/` and copies current
   tree in (recursive cp; on same filesystem this uses reflinks where supported).
2. Admin PUTs changed files, DELETEs removed ones — all into the staging dir.
3. `POST /_icd/deploys/<id>/commit` → server atomically renames the symlink at
   `/srv/current` to point at the new deploy.
4. Server keeps last 5 deploys for rollback, prunes older ones.

## Upgrading this server

When server.ts changes in a cms-admin release, users need to rebuild the
infrastructure (not just content-sync). The admin UI surfaces this via
`GET /_icd/health` — if `version` lags current, show "Rebuild infrastructure"
prompt. Content survives the rebuild (volume is persistent).
